# Markdown 文件转录语音

在给定文件同目录下创建 `audio` 文件夹作为存放生成文件的数据目录。

如果给的是纯文本就是当前目录下创建 `audio` 文件夹。

## 进入虚拟环境

1. 查看当前环境是否为 `conda` 的 `markdown-to-audio` 虚拟环境

```bash
conda env list
```

如果已经在虚拟环境中，跳过进入下一个二级标题内容

2. 查看是否存在该虚拟环境

如果上面命令返回的 `env` 中没有 `markdown-to-audio` 则首先创建

```bash
conda create --name markdown-to-audio python=3.10 -y
source /Users/wxnacy/.wzsh/plugins/wzsh-python/bin/conda-init && conda activate markdown-to-audio && pip install -U mlx-audio
```

命令中需要确认的步骤，自动确认

如果存在该 `env` 则坚决不要执行 `conda create` 命令

而是直接激活到该 `env`

```bash
source /Users/wxnacy/.wzsh/plugins/wzsh-python/bin/conda-init && conda activate markdown-to-audio
```

如果切换失败，程序退出，并通知我

## 清洗文本内容

将 `Markdown` 内容做清洗，大原则只保留正文的纯文本，细节如下

- 属性信息全部删除
- 标题全部删除（不管几级标题都删除）
- 删掉链接文本
- 删掉代码块

独立的段落要保留格式，两个段落间保留至少一个回车

## 拆分内容

以一个段落为一个最小单元，将全文拆分成多个片段，每个片段的字符数量在 `300` 之内。

一个片段可以是多个段落或一个段落组成，切记要保留完整的段落，如果一个段落的字符就超了 `300`，那这个片段就用这一个段落即可。

将这些片段以 `{序号:保留两位数}-{段落第一句话}.md` 的形式都保存到数据目录中。

## 转录声音

按照片段顺序逐个生成语音，使用如下脚本进行声音转录

```bash
source /Users/wxnacy/.wzsh/plugins/wzsh-python/bin/conda-init && conda activate markdown-to-audio && python -m mlx_audio.tts.generate \
  --model mlx-community/Qwen3-TTS-12Hz-1.7B-Base-bf16 \
  --lang_code zh\
  --text "{替换为需要转录的文本}" \
  --ref_audio "/Users/wxnacy/Downloads/MIShare/2026年01月26日 20点08分.m4a" \
  --ref_text "你好！我是来测试声音克隆效果的。2025年1月26日，今天天气真不错，温度大概18度左右。请问你现在几点了？哦，已经晚上七点三十八分了！我喜欢喝咖啡，尤其是拿铁加一点焦糖。Hello world! 这句话用英文说也挺有感觉的。人生就像一盒巧克力，你永远不知道下一颗是什么味道……哈哈，不过我更期待明天的惊喜！一二三四五六七八九十，ABCDEF，xyz。今天心情超级好～期待和你一起聊天！" \
```

本命令会把语音保存为 `audio_000.wav`，需要将文件重命名为片段 `md` 文件同名 `wav` 格式
